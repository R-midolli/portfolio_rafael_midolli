Architecture TechniqueTechnical Architecture

Ce projet de bout en bout utilise Pandas et l'API-Football pour ingérer des gigaoctets de données de matchs. Celles-ci sont transformées (scikit-learn) et stockées au format Parquet. Le backend exécute des modèles XGBoost / CatBoost pour prédire le risque de blessure et la performance. Le tout est exposé dans un dashboard Streamlit richement interactif avec des graphiques Plotly. This end-to-end project utilizes Pandas and API-Football to ingest gigabytes of match data. These are transformed (scikit-learn) and saved in Parquet format. The backend runs XGBoost / CatBoost models predicting injury risk and performance. Everything is showcased in a highly interactive Streamlit dashboard featuring Plotly charts.

🎯 Enseignements Clés & Recommandations 🎯 Key Takeaways & Recommendations

1. Centralisation des Couches de Données : Permettre de passer d'un affichage "Fan" (simplifié) à un affichage "Expert" (modèles SHAP, optimisation linéaire) démontre la flexibilité de Streamlit et l'importance d'adapter la donnée à son audience. 1. Centralizing Data Layers: Allowing the user to seamlessly switch from a "Fan" perspective (simplified) to an "Expert" perspective (SHAP models, linear optimization) demonstrates the flexibility of Streamlit and adapting data to the audience.

2. Prédiction des Blessures via ML : En injectant des métadonnées (âge, fatigue, historique) dans CatBoost, l'application anticipe visuellement le risque de blessure par jogador, agissant comme un véritable outil d'aide à la décision médicale et tactique. 2. Injury Prediction via ML: By injecting metadata (age, fatigue, history) into CatBoost, the application visually anticipates the injury risk per player, acting as a genuine medical and tactical decision-support tool.

3. Pipeline de test robuste : Avec l'utilisation de `pytest` automatisé via GitHub Actions, ce projet s'assure d'une couverture importante sur l'orchestration des données avant d'arriver au Dashboard. 3. Robust Testing Pipeline: By utilizing automated `pytest` via GitHub Actions, this project ensures high test coverage on data orchestration before allowing it into the Dashboard.
